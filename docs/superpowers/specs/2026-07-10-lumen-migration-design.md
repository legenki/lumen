# Lumen — дизайн переноса filtr-tool (2026-07-10)

## Цель

Перенести весь функционал старого инструмента **filtr-tool**
(https://antlii.github.io/filtr-tool/) в новое standalone-приложение **Lumen**
на дизайн-системе Grafema/Ritmo, по процессу, отработанному на divix
(см. `LUMEN_METHODOLOGY.md` и `AGENTS.md`). По ходу переноса — оптимизация
архитектуры и исправление ошибок старой реализации.

## Зафиксированные решения

| Вопрос | Решение |
| --- | --- |
| Исходники старого кода | Оригиналов нет; работаем с минифицированным Vite-бандлом с GitHub Pages (он хорошо читается: сохранены строки, метки, структура) |
| Объём переноса | Всё, **кроме системы лицензий** (противоречит offline-first PWA) |
| Критерий успеха | **Визуальный паритет пресетов**: старые пресеты дают тот же результат; шейдеры переносятся дословно |
| UI-компоновка | **Две панели**: слева — Presets/Media/Canvas/Export/Layers, справа — инспектор выбранного слоя |
| Подход | Реверс из бандла (A) + старый бандл как эталон для сравнения рендеров (C) |

## Инвентарь функционала старого инструмента (из бандла)

- **19 модулей эффектов**, собираемых в стек:
  - Fill: Media File, Color, Gradient, Noise Grain (`fillMedia`, `fillColor`, `fillGradient`, `fillNoise`)
  - Displace: Sine, Cubic Grid, Simplex Noise, Texture Map (`displaceSine`, `displaceCubic`, `displaceSimplex`, `displaceTexture`)
  - Blur: Gaussian, Motion, Blue-Noise (`blurGaussian`, `blurMotion`, `blurNoise`)
  - Color: Gradient Map, Color Adjust, RGB Shift, Luma Bands (`gradientMap`, `colorCorrection`, `rgbShift`, `lumaBands`)
  - Прочее: Emboss/Relief, Lens Grid, Warp Grid (`embossEffect`, `lensGrid`, `warpGrid`)
  - **MASK: Media File** (`maskMedia`, `type: "mask"`) — группирует другие слои
- **~28 встроенных пресетов** + user preset
- **WebGL2-пайплайн** из шейдерных пассов: 15 оригинальных `.frag`-файлов
  (GLSL ES 3.00, `#version 300 es`) — они деплоились отдельными файлами и
  сохранились в неминифицированном виде с комментариями; 26 blend modes
  (Photoshop-набор). Часть модулей (blurGaussian/blurMotion, displaceTexture,
  colorCorrection, maskMedia) переиспользует общие шейдеры — точная карта
  модуль→пассы составляется на этапе реверса
- **Медиа-менеджер**: фото и видео, слоты, seek по кадрам, выбор канала
- **Рекордер WebM** (webm-muxer): Animation Length/Range, FPS, Bitrate Level, Frame Quality
- UI старого — Tweakpane: «Stack Sequence» + отдельный инспектор
- Система лицензий — **не переносится**

### Семантика MASK (проверено по бандлу)

- У инстанса маски есть `__maskMembers: []` — **явный список id** дочерних слоёв
  (не «всё, что ниже по стеку»).
- Маска не может быть членом другой маски (в старом коде — предупреждение
  `member-is-mask … (ignored)`).
- Параметры маски: текстура (media, allowNone), Mask Threshold (interval),
  Use Channel, Invert Colors, Contrast (interval), Scale/Rotate/Position, Wrap Mode.
- Визуально: члены — с отступом «↳» и акцентной полосой; на маске — бейдж
  с количеством членов.

### Состояние (state.js)

- **Persisted** (LocalStorage + пресеты): `cnv` (ratio, разрешение),
  `stack` — упорядоченный массив `{ id, type, enabled, params }`
  (у `type: "mask"` дополнительно `maskMembers: []`), настройки `rec`,
  ссылки на медиа-слоты.
- **Runtime**: p5.Graphics-буферы, скомпилированные шейдеры, video-элементы,
  инстансы шума.
- Стек — ядро модели: UI, пайплайн и пресеты рендерятся из него.
  Пресет = сериализованный `{ cnv, stack }` (+ версия схемы).

### Рендер-пайплайн (pipeline.js)

- Ping-pong между двумя offscreen `p5.Graphics(WEBGL)` фиксированного
  разрешения; модуль = один+ шейдерный пасс; выход пасса — вход следующего.
- Blend modes (26) — в шейдере, дословный перенос.
- Члены маски рендерятся в отдельную ветку и подмешиваются шейдером маски.
- **Пересборка по требованию**: пассы пересобираются только при изменении
  состава стека; кадр рендерится только при изменении параметров или активной
  анимации (dirty-flag + `noLoop`/`redraw`). Главная оптимизация против
  постоянного draw-цикла старого инструмента.
- Экранный канвас — 2D на всё окно; буфер рисуется `p.image()`
  с центрированием в рабочей области **между двумя панелями** и вписыванием
  в 85% доступного пространства.
- После любого `resizeCanvas()` у буферов — обязательное восстановление
  режимов (`angleMode`, `rectMode`, `strokeWeight`, `pixelDensity`).
- Zero-allocation в draw: пулы, переиспользование uniform-объектов,
  очистка массивов через `.length = 0`.

### UI: две панели (glassmorphism Grafema/Ritmo)

**Левая панель:**
- **Presets** — 28 встроенных + user-пресеты (save/load/import/export JSON, presetIO);
- **Media** — загрузка фото/видео, слоты, seek по кадрам, Use Channel;
- **Canvas** — ratio, разрешение, инфо;
- **Export** — PNG (high-res через `withHighResExport`) и WebM-рекордер;
- **Layers** — список слоёв стека:
  - дропдаун Module List (19 типов) + кнопка «Add To Stack»;
  - строки: drag-handle, имя; по ховеру — глаз (вкл/выкл), дублировать, удалить;
  - MASK-группы: отступ «↳», акцентная полоса, бейдж с числом членов;
    drag внутрь/наружу группы меняет `maskMembers`;
  - клик по слою — выделение → параметры в правой панели.

**Правая панель — инспектор** выбранного слоя: контролы декларативно
из `controls.js` через panelBuilder.

**Новые компоненты для Grafema** (разрабатываются в
`/Users/andy/Documents/GitHub/grafema`, стиль Glassmorphism, `--margin-xlarge`):
1. **LayerList** — карточки с drag-reorder, группировкой (маски), бейджами,
   hover-действиями, выделением — самый сложный новый компонент;
2. **CenterPoint** — 2D-джойстик (Center Point / Focus Center / Position Offset);
3. **GradientMapper** — многоточечный редактор градиента (Fill: Gradient,
   Gradient Map).

Остальные параметры покрываются существующими контролами (slider, select,
color, angle, toggle, interval). Точный аудит контролов — на этапе
имплементационного плана, после разбора всех 19 модулей.

### Рандомизация и персистенция

- `randomize.js` — строго посекционно: randomize модуля мутирует только его
  `params`; сиды шума и цвета не трогаются при рандомизации трансформаций.
- Автосохранение persisted-состояния в LocalStorage; PWA-манифест;
  все ассеты локальные.

### Обработка ошибок

- Медиа: уведомления при неподдерживаемом/битом файле, слот откатывается
  на дефолт.
- WebGL: проверка компиляции шейдеров на старте; понятное сообщение,
  если WebGL недоступен.
- Пресеты: валидация структуры при импорте JSON; неизвестные поля
  игнорируются; версия схемы для будущих миграций.

## Тестирование и верификация паритета

- **vitest**: сериализация/миграция пресетов, схемы параметров модулей,
  утилиты state, логика maskMembers (перемещение слоёв в/из группы).
- **Эталон**: старый бандл в `reference/filtr/live/` запускается локально;
  после переноса каждого модуля рендеры пресетов сравниваются со старым
  инструментом на одинаковых входных изображениях.
- Критерий приёмки пресета: визуально неотличимый результат (шейдеры
  дословные; расхождения возможны только из-за порядка пассов/uniform'ов —
  сравнение их выявляет).

## Вне объёма

- Система лицензий (проверка ключей, статусы подписки).
- Пиксельно-точное побайтовое совпадение экспорта (критерий — визуальный).
- Миграция user-пресетов из LocalStorage старого инструмента (другой домен;
  при необходимости — импорт JSON вручную).
